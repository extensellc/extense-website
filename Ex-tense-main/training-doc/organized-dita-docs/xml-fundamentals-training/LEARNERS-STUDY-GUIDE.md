# XML Fundamentals — Learner's Study Guide

## Overview

This study guide helps you plan your learning journey through the XML Fundamentals Training Guide. Work through the phases at your own pace — each phase builds on the previous one.

---

## Phase 1: XML Basics (Days 1-2)

**Goal:** Write well-formed XML documents from scratch.

### Read
- Chapter 1: What Is XML?
- Chapter 2: XML Syntax Rules
- Chapter 3: Elements and Attributes
- Chapter 4: XML Namespaces
- XML Quick Reference (keep open as a cheat sheet)

### Practice
- Complete **Exercise 1: Writing Well-Formed XML Documents**
- Create at least 3 XML documents of your own choosing

### Self-Assessment
- [ ] I can write a well-formed XML document without looking at examples
- [ ] I can identify and fix all common XML syntax errors
- [ ] I can explain the XML document tree (parent, child, sibling, ancestor)
- [ ] I can declare and use namespaces
- [ ] I know what entity references are and when to use them

---

## Phase 2: DTD Validation (Days 3-4)

**Goal:** Validate XML documents against DTD schemas.

### Read
- Chapter 5: What Is a DTD?
- Chapter 6: DTD Element Declarations
- Chapter 7: DTD Attributes and Entities
- DTD Quick Reference

### Practice
- Complete **Exercise 2: Writing DTDs and Validating XML**
- Write a DTD for one of your own XML documents from Phase 1

### Self-Assessment
- [ ] I can explain the difference between well-formed and valid XML
- [ ] I can write element declarations with sequences and occurrence indicators
- [ ] I can write attribute declarations with types and defaults
- [ ] I can create both internal and external DTDs
- [ ] I can define and use entities for reusable content

---

## Phase 3: XSD Schemas (Days 5-7)

**Goal:** Write XSD schemas with data types and restrictions.

### Read
- Chapter 8: What Is XSD?
- Chapter 9: XSD Elements and Types
- Chapter 10: XSD Restrictions and Patterns
- XSD Quick Reference

### Practice
- Complete **Exercise 3: Writing XML Schemas (XSD)**
- Convert your Phase 2 DTD to an equivalent XSD — note what you can express in XSD that you couldn't in DTD

### Self-Assessment
- [ ] I can write XSD schemas with simple and complex types
- [ ] I can use built-in data types (string, integer, decimal, date, boolean)
- [ ] I can create custom restricted types with facets and patterns
- [ ] I can define named complex types and reuse them
- [ ] I can extend a complex type (type inheritance)
- [ ] I can explain when to choose DTD vs. XSD

---

## Phase 4: DITA Introduction (Days 8-10)

**Goal:** Create a complete DITA mini-project with concept, task, and reference topics.

### Read
- Chapter 11: What Is DITA?
- Chapter 12: DITA Topic Types
- Chapter 13: DITA Maps and Publishing
- DITA Element Reference

### Practice
- Complete **Exercise 4: Your First DITA Project**
- Build HTML5 output and browse the result

### Self-Assessment
- [ ] I can create valid concept, task, and reference topics
- [ ] I can choose the right topic type for different content
- [ ] I can assemble topics into a DITA map
- [ ] I can build output with DITA-OT
- [ ] I can explain how DITA uses XML, DTDs, and validation
- [ ] I understand what content reuse means in DITA (conref, keyref, conditions)

## Tips for Success

1. **Type the examples** — don't just read them. Muscle memory matters for XML syntax.
2. **Break things on purpose** — deliberately introduce errors and learn to read error messages.
3. **Use the reference topics** — they exist so you don't have to memorize syntax.
4. **Validate constantly** — run validation after every change, not just at the end.
5. **Connect the concepts** — every DITA feature maps back to an XML, DTD, or XSD concept you learned.
