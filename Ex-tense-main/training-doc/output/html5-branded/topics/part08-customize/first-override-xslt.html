Write your first override XSLT correctly

Import the default HTML5 implementation

Your override stylesheet should start by importing the default HTML5 transform:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
                version="3.0">

  <xsl:import href="plugin:org.dita.html5:xsl/dita2html5.xsl"/>

  <!-- overrides below -->

</xsl:stylesheet>

Precedence rules (do not skip)

  • templates in your override file win over imported templates
  • within the same precedence level, more specific matches win
  • import order matters if you have multiple imports

Practical implication: keep overrides in a single file (or layered imports you control) and avoid rewriting entire default templates unless necessary.

Prefer xsl:next-match for safe layering

When overriding a template but wanting default behavior, wrap with xsl:next-match:

<xsl:template match="*[contains(@class,' topic/table ')]">
  <div class="table-wrap">
    <xsl:next-match/>
  </div>
</xsl:template>

This is safer than copy-pasting large default templates.
