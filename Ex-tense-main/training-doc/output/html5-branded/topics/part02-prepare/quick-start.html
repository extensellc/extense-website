Quick Start Guide

30-Minute Fast Track

Step 1: Understand What DITA Is (10 minutes)

Read: DITA Quick Introduction Guide

Key takeaways:

  • DITA = structured XML content that's reusable and multi-format
  • Topics are small, focused information units
  • Maps assemble topics into deliverables
  • Publishing happens through DITA-OT

Skip to: Section 2 (DITA mental model) and Section 4 (most important features)

Step 2: See a Working Example (10 minutes)

Explore: the examples/dita-html5-sidebar-starter-v2/ directory in the starter repo

Actions:

  1. Look at sample topics in dita/topics/
  2. Open the map in dita/maps/getting-started.ditamap
  3. Review the plug-in structure in plugins/

What you'll learn:

  • What a DITA topic looks like
  • How maps organize topics
  • Basic project structure

Step 3: Try Your First Build (10 minutes)

Prerequisites: DITA-OT 4.x installed (dita-ot.org)

Build command:

cd examples/dita-html5-sidebar-starter-v2
./build/build.sh  # or build.ps1 on Windows

View output: Open out/html/index.html in browser

What you'll learn:

  • How to run a DITA build
  • What HTML output looks like
  • Where files are generated

Common Questions Answered

"What's the difference between DITA and Markdown/Word/etc?"

Answer: Getting Started → Section 1

DITA is structured, semantic, and designed for:

  • Content reuse at scale
  • Multiple output formats from one source
  • Product/platform/audience variants
  • Long-lived documentation with frequent updates

"How do I know if my team needs DITA?"

Answer: DITA Overview → When to use DITA

Consider DITA if you have:

  • Multiple products/variants needing shared content
  • Multiple output formats (HTML + PDF + help)
  • Large documentation sets (100+ topics)
  • Frequent releases and updates
  • Multiple writers needing consistency

Skip DITA if you have:

  • Single product, single format
  • Small doc set (<20 topics)
  • Infrequent updates
  • No reuse requirements

"How long does it take to learn DITA?"

Answer: Depends on role and depth

  • Basic authoring: 1-2 weeks
  • Publishing/variants: 2-4 weeks
  • Development/customization: 4-8 weeks

Use learning paths above to accelerate.

"What tools do I need?"

Answer: Publishing Guide → Tools

Required:

  • DITA-OT (free, open-source)
  • XML editor (Oxygen, VS Code, or text editor)
  • Java 17+ (for DITA-OT)

Optional:

  • CCMS (enterprise content management)
  • Docker (for builds)
  • Git (for version control)

"Can I migrate existing docs to DITA?"

Answer: Yes, common approaches:

  1. Manual conversion (highest quality, most effort)
  2. Scripted conversion (Markdown/HTML → DITA)
  3. Gradual migration (new content in DITA)
  4. Hybrid approach (convert high-value content first)

See Developer Handbook for migration strategies.

"How do I customize HTML output?"

Answer: XSLT Customization Tutorial

Two safe approaches:

  1. CSS only (styling, no structure changes)
  2. XSLT-import overrides (structure + styling)

Avoid modifying core DITA-OT files.

Troubleshooting First Steps

Build fails?

  1. Check DITA-OT installation (dita --version)
  2. Verify Java version (java -version → 17+)
  3. Review error message for specific issue
  4. Check Architecture Diagram to identify stage

Content not appearing?

  1. Check DITAVAL filtering (conditional processing)
  2. Verify topic is referenced in map
  3. Check chunking settings
  4. Examine temp files in work directory

HTML looks wrong?

  1. Verify CSS path in build properties
  2. Check XSLT overrides are installed
  3. Clear browser cache
  4. Review generated HTML source
