Example 1: Concept topic — Explaining content reuse

BEFORE — Markdown:

# DITA Reuse and Variants

## Overview
Content reuse is a key benefit of DITA. You can reuse content
at multiple levels using different mechanisms.

## Conref
The conref mechanism lets you pull content from one topic into
another. It works like a symlink for content.

Example:
```xml
<p conref="shared/warnings.dita#warnings/safety-warning"/>
```

## Keyref
Keys provide indirect references. Define a key in your map,
then reference it in topics.

## DITAVAL Filtering
Use DITAVAL files to include or exclude content based on
audience, platform, or product attributes.

## Best Practices
- Always use keys for product names
- Prefer keyref over conref when possible
- Test all variants before publishing

What changed and why:

Change made Why it was needed AI/chatbot impact
File typed as <concept> This content explains ideas, not procedures AI formats answer as explanation, not steps
Added <shortdesc> Serves as chunk summary in vector search Every chunk inherits this summary for context
Added 5 <indexterm>s including "Conditional processing" Users search by multiple terms for the same concept Both "DITAVAL" and "conditional processing" queries find this topic
Set content-domain="content-reuse" Classifies by subject area Domain-filtered retrieval: only return reuse content when user asks about reuse
Set content-intent="explain" Declares this explains a concept, not instructs a task "What is conref?" routes here; "How do I set up conref?" routes to a task topic

Example 2: Task topic — Installing DITA-OT

BEFORE — Markdown:

# Installing DITA-OT

You need Java 17+ installed first.

1. Go to https://www.dita-ot.org/download
2. Download the latest .zip release
3. Unzip to a directory (e.g., ~/dita-ot)
4. Add ~/dita-ot/bin to your PATH
5. Test: run `dita --version`

If you see the version number, you're all set!

AFTER — AI-Ready DITA task:

<task id="installing-dita-ot" xml:lang="en-US">
  <title>Installing DITA-OT</title>
  <shortdesc>Download, install, and verify DITA Open Toolkit
    4.x on your machine.</shortdesc>
  <prolog>
    <!-- Full AI-ready metadata... -->
    <data name="difficulty" value="beginner"/>
    <data name="content-intent" value="how-to"/>
    <data name="chatbot-priority" value="high"/>
    <data name="prerequisites" value="none"/>
  </prolog>
  <taskbody>
    <prereq>
      <ul>
        <li><b>Java 17 or later</b> — Check with
          <codeph>java -version</codeph>.</li>
        <li><b>500 MB free disk space</b></li>
        <li><b>Network access</b> to download</li>
      </ul>
    </prereq>
    <steps>
      <step>
        <cmd>Download the latest release from
          dita-ot.org/download</cmd>
        <info>Choose the .zip distribution for your
          platform.</info>
      </step>
      <step>
        <cmd>Unzip to your preferred directory.</cmd>
        <stepxmp>
          <codeblock outputclass="language-bash">
unzip dita-ot-4.2.zip -d ~/
          </codeblock>
        </stepxmp>
        <stepresult>A dita-ot-4.2/ directory is
          created.</stepresult>
      </step>
      <step>
        <cmd>Add the bin/ directory to your PATH.</cmd>
        <stepxmp>
          <codeblock outputclass="language-bash">
export PATH=~/dita-ot-4.2/bin:$PATH
          </codeblock>
        </stepxmp>
      </step>
    </steps>
    <result>
      <p>Run <codeph>dita --version</codeph>. You should
        see: <codeph>DITA-OT version 4.2</codeph>.</p>
    </result>
  </taskbody>
</task>

Key improvements:

  • Prerequisites are semantically separated — AI can show them before the steps, and warn users if prerequisites are not met
  • Steps use <cmd> — the imperative instruction is clearly separated from supporting info, examples, and results
  • <stepresult> — AI can tell users what to expect after each step
  • <result> — AI can describe the expected outcome to help users verify success
  • content-intent="how-to" — "How do I install DITA-OT?" queries are routed directly here

What the chatbot can now do:

  • Filter for audience=developer — show this only to developers, not content strategists
  • Filter for difficulty=intermediate — don't show to absolute beginners
  • Route content-domain=devops — only return when user asks about devops topics
  • Match content-intent=explain — show for "what is..." questions, not "how to..." questions
  • Apply chatbot-priority=high — rank above low-priority reference material
  • Suggest prerequisites=dita-publishing — "You should read DITA Publishing first"
  • Use critdates — rank fresher content higher
